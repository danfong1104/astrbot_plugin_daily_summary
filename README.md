# 每日群聊总结插件

自动总结群聊消息，统计活跃度，提取话题，生成精彩回顾。

## 功能特性

- 📊 **消息统计**：统计群聊消息总数，按活跃度排名
- 🏆 **排行榜**：显示最活跃的群友前三名
- 💡 **话题提取**：AI自动提取今日讨论的主要话题
- ✨ **精彩瞬间**：提取群聊中的有趣亮点
- 📝 **整体总结**：50字以内总结一整天的聊天氛围
- ⏰ **定时推送**：每天自动推送总结到指定群聊
- 🎯 **手动触发**：支持 `/总结今日` 和 `/总结昨日` 命令
- 🔧 **灵活配置**：支持中英文冒号、昵称映射等自定义设置

## 安装方法

1. 将插件文件夹 `astrbot_plugin_daily_summary` 放入 AstrBot 的 `data/plugins/` 目录
2. 重启 AstrBot 或在 WebUI 中重新加载插件
3. 在插件配置中设置相关参数

## 配置说明

在 AstrBot WebUI 的插件配置页面中，可以设置以下参数：

### 基础配置

| 配置项 | 说明 | 默认值 | 示例 |
|--------|------|--------|------|
| 报告类型 | 选择总结今日或昨日消息 | daily | daily/yesterday |
| 推送时间 | 每日自动推送的时间 | 23:00 | 23:00 或 23：00 |
| 推送群聊 | 需要推送总结的群号列表 | [] | [123456789, 987654321] |
| 昵称名单 | QQ号到昵称的映射字典 | {} | {"1234567": "小明"} |
| 最大字数 | 总结报告的最大字数限制 | 1000 | 500-2000 |
| 调试模式 | 开启详细调试日志 | false | true/false |

### 配置示例

```json
{
  "report_type": "daily",
  "push_time": "23:00",
  "group_ids": [123456789, 987654321],
  "nickname_mapping": {
    "1234567": "小明",
    "7654321": "小红",
    "1111111": "技术大牛"
  },
  "max_length": 1000,
  "debug_mode": false
}
```

### 中英文冒号支持

配置项支持中英文冒号，系统会自动处理：
- `23:00` 或 `23：00` 都可以
- 昵称映射中的键值对也支持中英文冒号

## 使用方法

### 手动命令

- `/总结今日 [群号]` - 立即生成今日群聊总结
- `/总结昨日 [群号]` - 立即生成昨日群聊总结
- `/summary_help` - 显示帮助信息

### 使用示例

```
/总结今日 123456789    # 总结群号123456789的今日消息
/总结昨日              # 总结当前群聊的昨日消息
/总结今日              # 总结当前群聊的今日消息
```

### 自动推送

在配置中设置 `group_ids` 列表，插件会在每天指定时间自动推送总结到这些群聊。

## 注意事项

1. **平台支持**：确保 AstrBot 已接入相应的聊天平台（如 QQ、微信等）
2. **消息记录**：群聊需要有足够的消息记录才能生成有意义的总结
3. **LLM配置**：AI总结功能需要配置 LLM 提供商（如 OpenAI、DeepSeek 等）
4. **测试建议**：建议在群聊中先测试手动命令，确认功能正常后再开启自动推送
5. **权限要求**：机器人需要有发送消息的权限
6. **数据存储**：插件使用 AstrBot 的对话管理器获取历史消息，无需额外存储

## 故障排除

### 无法获取历史消息

1. 检查群号是否正确
2. 确认机器人在群内且有读取消息的权限
3. 开启调试模式查看详细日志

### AI总结失败

1. 检查 LLM 提供商是否配置正确
2. 确认 API 密钥有效
3. 查看日志中的错误信息

### 定时推送不工作

1. 检查推送时间格式是否正确（支持中英文冒号）
2. 确认群号列表不为空
3. 查看插件是否正常加载

## 依赖要求

- httpx（用于异步网络请求）
- apscheduler（用于定时任务，可选）
