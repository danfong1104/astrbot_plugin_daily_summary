# 每日群聊总结 (Daily Summary)

基于 AstrBot 的群聊每日总结插件，自动统计群聊活跃度、AI 提炼话题与金句，定时推送到指定群聊。

---

## 初衷

市面上大多数群聊日报插件都倾向于把内容渲染成图片，乍一看挺酷，但执行久了群友都懒得点开——毕竟要多一步"点开大图"的操作，在刷屏的聊天流里天然被跳过。

**纯文字反而更有阅读体验。** 不用点开、不用加载，一眼扫完。OneBot 11 的纯文本不支持 Markdown，所以在排版上花了不少心思：用 emoji 做视觉分区、用 🟢 前缀标识话题条目、用 🥇🥈🥉 排活跃度——在不依赖任何富文本的情况下，让日报读起来清爽舒服。

目前调试效果已经相当不错了，欢迎大家体验。

---

## 功能特性

- **消息统计**：统计当日/昨日群聊消息总数，展示活跃度前三名
- **AI 话题总结**：调用大模型提炼群聊讨论话题，支持上下文关联归纳
- **今日金句**：自动提取群聊中最有意思、最精辟的发言
- **整体总结**：一句话概括群聊氛围
- **定时推送**：每天指定时间自动推送到配置的群聊
- **手动触发**：支持 `/总结今日`、`/总结昨日` 命令随时生成
- **排除用户**：支持排除指定 QQ 号的消息（如机器人自己）
- **昵称映射**：支持配置 QQ 号到显示昵称的映射
- **可调节字数**：话题描述、整体总结的字数上限均可独立配置

## 安装

在 AstrBot 插件市场中搜索 `astrbot_plugin_daily_summary` 安装，或手动将插件文件夹放入 `data/plugins/` 目录。

### 依赖

插件会自动使用 AstrBot 已配置的 LLM 模型，无需额外配置。可选依赖：

- `apscheduler`：用于定时任务调度（未安装时自动降级为内置简单定时器）

---

## 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `report_type` | 下拉选择 | `daily` | 报告类型：`daily`（今日）/ `yesterday`（昨日） |
| `push_time` | 文本 | `23:00` | 每日自动推送时间，格式 `HH:MM` |
| `group_ids` | 列表 | `[]` | 需要推送的群号列表 |
| `nickname_mapping` | 多行文本 | — | QQ 号到昵称的映射，每行格式：`QQ号:昵称` |
| `exclude_ids` | 多行文本 | — | 不纳入统计的 QQ 号，每行一个（如机器人 QQ） |
| `summary_prompt` | 多行文本 | 轻松幽默 | 总结的口吻风格，如"正式严肃""可爱俏皮" |
| `max_messages` | 整数 | `200` | 用于生成总结的历史消息条数，越大覆盖话题越全 |
| `topic_max_length` | 整数 | `30` | 每条话题描述的最大字数 |
| `summary_max_length` | 整数 | `80` | 整体总结部分的最大字数 |
| `max_length` | 整数 | `1000` | 整个日报的最大字数限制 |
| `debug_mode` | 布尔 | `false` | 开启调试日志 |

## 手动命令

| 命令 | 说明 |
|------|------|
| `/总结今日 [群号]` | 立即生成本群/指定群的今日总结 |
| `/总结昨日 [群号]` | 立即生成本群/指定群的昨日总结 |
| `/summary_help` | 显示帮助信息 |

## 话题提炼逻辑

插件通过精心设计的 prompt 驱动 LLM 进行话题总结，核心规则：

1. **上下文关联**：不会孤立提取关键词，而是联系上下文归纳——例如"面具能骗过扫脸识别"会归入"高考安检方式讨论"
2. **按热度排序**：消息数多、参与人数多的话题优先收录
3. **智能合并**：超过 5 个独立话题时自动归类合并，一条描述涵盖多个相关讨论
4. **过滤噪音**：一两个人草草聊两句的话题不会收录
5. **时间顺序**：话题按聊天中的讨论先后排列

---

## 技术说明

- 基于 OneBot 11 协议的 `get_group_msg_history` API 获取历史消息
- 消息按时间正序排列后送入 LLM，确保总结的时间线准确
- AstrBot v4 适配：自动获取平台适配器 ID 构造 `unified_msg_origin`
- APScheduler 可用时优先使用，否则降级为内置 asyncio 定时器
